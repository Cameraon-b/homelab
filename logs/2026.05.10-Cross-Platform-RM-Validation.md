# Entry 04 — Cross-Platform SSH Enablement, Bootloader Configuration, and Multi-Node Administration

## 1. Reeba Dual-Boot Recovery and GRUB Configuration

### Issues Encountered

During Windows Update:

- Automatic reboot redirected system into Kali Linux
- GRUB defaulted to Kali
- Temporary read-only filesystem state observed after unclean shutdown

### Resolution

Booted into Kali.

Validated GRUB entries.

Regenerated boot entries:

```bash
sudo update-grub
```

Confirmed:

- Windows Boot Manager rediscovered

Modified GRUB default entry to:

- Microsoft Windows Boot Manager

### Result

Subsequent Windows update cycles rebooted directly back into Windows without manual intervention.

## 2. OpenSSH Server Deployment on Reeba

### Issues Encountered

Initial SSH attempts from Nora and mobile clients resulted in:

- Connection timeout
- `sshd` service missing

### Root Cause

Only OpenSSH Client was installed.

### Resolution

Installed:

- OpenSSH Server

Started service:

```powershell
Start-Service sshd
Set-Service -Name sshd -StartupType Automatic
```

Validated service state:

```powershell
Get-Service sshd
```

Validated port listening:

```powershell
netstat -ano | findstr :22
```

### Result

```text
0.0.0.0:22 LISTENING
```

---

## 3. Firewall Profile Troubleshooting

### Issues Encountered

SSH handshake stalled despite:

- Port 22 listening
- Firewall rule present

### Root Cause

- Windows network profile = Public
- SSH firewall rule = Private only

### Resolution

Modified firewall rule:

```powershell
Set-NetFirewallRule -Name OpenSSH-Server-In-TCP -Profile Any
```

Restarted SSH service.

### Result

Remote SSH immediately succeeded.

## 4. Authentication Troubleshooting

### Issues Encountered

### Root Cause

