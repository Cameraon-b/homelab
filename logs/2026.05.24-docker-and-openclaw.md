### Docker Volumes
Created a persistent Docker volume:

```bash
sudo docker volume create aether-web-data
```

Attached it to nginx:

```bash
sudo docker run -d \
  --name aether-nginx \
  -p 8080:80 \
  -v aether-web-data:/usr/share/nginx/html \
  nginx
```

Edited the nginx `index.html` directly in the volume path:

```bash
sudo nano /var/lib/docker/volumes/aether-web-data/_data/index.html
```

Confirmed that custom content persisted after stopping, removing, and recreating the container.

Key concept learned:

```text
Containers are disposable.
Volumes preserve important data.
```

### Docker Compose
Created a Compose project for nginx:

```bash
mkdir ~/aether-compose-lab
cd ~/aether-compose-lab
nano docker-compose.yml
```

Defined:

- nginx service
- persistent volume
- custom Docker network
- restart policy

Initial issue:

Docker Compose created a project-prefixed volume instead of using the manually created volume.

Resolution:

Marked the existing volume as external:

```yaml
volumes:
  aether-web-data:
    external: true
```

Practiced Compose commands:

```bash
sudo docker compose up -d
sudo docker compose down
sudo docker compose ps
sudo docker compose logs
```

Key concept learned:

```text
Docker Compose defines application infrastructure in YAML rather than long command-line syntax.
```

### Monitoring Troubleshooting

#### Cockpit SSL Warning
Issue:

```text
Self-signed certificate warning
```

Resolution:

Configured Kuma to ignore TLS/SSL validation for the internal Cockpit monitor.

---

#### Windows Ping Failure
Issue:

Reeba was online but did not respond to ping from Kuma.

Resolution:

Enabled ICMP Echo Request in Windows Firewall:

```powershell
Enable-NetFirewallRule -Name FPS-ICMP4-ERQ-In
```

Confirmed immediate successful ping responses from Nora and green status in Kuma.

---

#### DNS Monitor Configuration
Initially struggled with DNS monitor field configuration.

Correct configuration used:

```text
Hostname: domain record to resolve
DNS Server: WIN-DC01 IP
Port: 53
Record Type: A
```

Confirmed successful DNS health checks through Kuma.

## Issues Encountered

### Minimal Container Tools
Inside nginx container:

```text
ip: command not found
```

Cause:

Official nginx image is intentionally minimal and does not include `iproute2`.

Resolution:

Used:

```bash
sudo docker inspect
```

from the host instead.

---

### Compose Volume Naming
Compose created a project-scoped volume instead of using the manually created Docker volume.

Resolution:

Declared the volume as external.

---

### Windows Firewall Blocking Ping
Kuma could not ping Reeba even though the system was online.

Resolution:

Enabled ICMP Echo Request in Windows Firewall.

### Cockpit Self-Signed Certificate

## Key Takeaways

- Docker containers are isolated application processes, not full virtual machines.
- Docker images act as reusable blueprints.
- Volumes preserve persistent data independently of containers.
- Docker networks provide isolated communication between services.
- Docker Compose defines services and infrastructure in YAML.
- Compose projects are directory-based unless a file path is explicitly specified.
- Monitoring creates visibility into infrastructure health.
- Windows Firewall can block ping even when systems are online.
- Nora is now functioning as both:
  - virtualization host
  - container platform
