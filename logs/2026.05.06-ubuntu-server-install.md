## Attempt 1

### Observations

Installer detected USB Wi-Fi adapter automatically and loaded drivers.

### Notable Discovery

Previous Linux desktop installations required manual Wi-Fi driver installation.

Ubuntu Server automatically recognized the adapter during installation.

### Issue Encountered

Installer hung at:

```text
subiquity/late/apply config
```

### System Behavior

Observed:

- Keyboard remained responsive
- USB activity lights continued flashing
- No installer progress

### Action Taken

System left running overnight for additional observation.

### Result

No change after approximately 8 hours.

### Initial Hypotheses

Possible causes:

- USB installation media issue
- USB Wi-Fi adapter driver/configuration conflict

---

## Attempt 2

### Changes Made

- Switched to 32GB PNY USB media
- Removed USB Wi-Fi adapter
- Ethernet-only installation

### Result

Installation completed successfully.

---

## Preliminary Conclusion

Original installation failure was likely caused by one of the following:

- Faulty or inconsistent USB installation media
- USB Wi-Fi adapter interaction during late-stage installer configuration

---

## Lessons Learned

- Modern Linux server distributions may automatically detect hardware previously requiring manual drivers
- Peripheral devices can affect unattended installer stages
- Leaving a stalled installer running can help distinguish slow operations from hard hangs
- Simplifying hardware during installation reduces troubleshooting variables
