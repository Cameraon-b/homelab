## 1. Windows 11 Client Deployment Failure Analysis

Attempted deployment of:

**Windows 11 Enterprise Evaluation**

using multiple VM configurations.

### Configurations Tested

* UEFI boot via TianoCore / OVMF
* Legacy BIOS boot
* Secure Boot enabled / disabled
* VirtIO disk
* SATA disk
* Multiple boot order modifications
* Manual firmware XML modifications

---

## Symptoms Observed

* ISO detected by firmware
* DVD boot entry visible
* Selecting DVD resulted in black screen
* Guest returned to firmware boot selection
* Installer never launched

---

## Validation Performed

Verified ISO integrity:

### File size

`6.7 GB`

### File type

`ISO 9660 CD-ROM filesystem (bootable)`

### SHA-256 hash generated prior to transfer

Validated SCP transfer integrity.

---

## Root Cause Conclusion

Likely compatibility issue between:

**Windows 11 Enterprise Evaluation media**

and current QEMU / firmware stack.

### Resolution

Pivoted to:

**Windows 10 Enterprise x64 Evaluation**

`127.0.0.1`

OpenSSH tunnel

```bash
ssh -L 5901:127.0.0.1:5901 nora
```

# 4. Domain Join Failure Analysis

Initial domain join attempts failed.

### Symptoms

* `aether.lab` unresolved
* Domain controller not discoverable
* Authentication prompt inconsistent

---

## Investigation

Validated:

### Network connectivity

`ping 192.168.1.15`

Successful.

### DNS configuration

Discovered client was using router DNS:

`192.168.1.1`

instead of domain controller DNS.

---

# 5. DNS Remediation

Modified client network adapter.

### Previous DNS

`192.168.1.1`

### Updated DNS

`192.168.1.15`

---

## Additional Discovery

Windows preferred IPv6 DNS from ISP.

### Resolution

Disabled IPv6 on `WIN-CLIENT01`.

---

## Validation

Executed:

```cmd
nslookup aether.lab
```

### Result

Successfully resolved:

`aether.lab → 192.168.1.15`

# Lessons Learned

* Guest OS media compatibility can vary across virtualization stacks.
* SHA-256 hashing validates installation media integrity.
* Simplifying VM configuration can isolate media-specific failures.
* Active Directory is highly dependent on DNS correctness.
* Windows may prioritize IPv6 DNS over IPv4 domain DNS.
* Organizational Units provide structure; groups provide permissions.
* Separate standard and privileged accounts mirror enterprise security practices.
