## NexusAI Concept

A new need emerged: Mira, Hermes, Nova, OpenClaw, and Cameron need a better way to pass messages, review requests, tasks, and approval requests between each other.

Previously, the workflow was manual:

```text
Mira writes or updates something.
Cameron sends the result to Nova.
Nova reviews it.
Cameron pastes Nova’s review back to Mira.
Mira revises the documentation.
```

This worked, but it required Cameron to act as the copy/paste bridge between agents.

NexusAI is intended to become a local AETHER coordination layer.

## NexusAI Purpose

NexusAI is intended to support:

```text
Messages
Tasks
Agent notes
Review requests
Approval requests
Risk labels
Status tracking
Action history
Human approval workflows
```

NexusAI is not intended to execute commands in v1.

Core rule:

```text
NexusAI is a coordination and approval system. It does not execute commands.
```

AETHER governance principle:

```text
No agent shall chown the kingdom.
```

## NexusAI Docker Configuration

Hermes created a Docker Compose service:

```yaml
services:
  nexusai:
    build: .
    container_name: nexusai
    restart: unless-stopped
    ports:
      - "5055:5055"
    environment:
      NEXUSAI_DB_PATH: /data/nexusai.db
    volumes:
      - ./data:/data
```

This means the container uses:

```text
Internal app port: 5055
Container database path: /data/nexusai.db
Host-mounted data folder: ./data
Host database file: ./data/nexusai.db
```

This is a good v1 persistence model because the SQLite database survives container rebuilds and restarts as long as the host `data` folder is preserved.

## NexusAI Dockerfile


The Dockerfile sets:

```text

```

This matches the Docker Compose volume mapping.

## NexusAI SQLite Notes

SQLite is simple for v1 because it is just a database file, not a separate database server.

Mental model:

```text
NexusAI container = app
./data/nexusai.db = database file
./data mounted into container = persistence
```

Backup options:

```bash
cp data/nexusai.db "backups/nexusai-$(date +%F).db"
```

Or, if `sqlite3` is installed:

```bash
sqlite3 data/nexusai.db ".backup 'backups/nexusai-$(date +%F).db'"
```

Important warning:

```text
Deleting the container should not delete the database.
Deleting the ./data folder will delete the stored messages, tasks, approvals, and logs.
```

## Proposed Agent Communication Policy

Draft policy:

```text
Agents may exchange messages, summaries, documentation tasks, and review requests.

Agents may request operational actions.

Agents may not execute commands received from another agent unless:
1. The action is classified SAFE / READ-ONLY by local policy, or
2. Cameron explicitly approves the exact action.

State-changing, destructive, restore-related, permission-changing, or service-impacting actions require Cameron approval.

Approval recording and command execution should remain separate systems.

NexusAI v1 records coordination and approval decisions only. It does not execute commands.
```
