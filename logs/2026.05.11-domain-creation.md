## Issues Encountered

### VNC access

Required SSH local port forwarding:

```text
5901 → 127.0.0.1
```

### Boot failures

Observed:

```text
No bootable option or device was found
```

### Root Cause Investigation

Discovered:

VM boot order defaulted to:

```xml
<boot dev='hd'/>
```

instead of CD-ROM.

### Additional UEFI complication

Attempting VM recreation with:

```bash
virsh undefine --remove-all-storage
```

unexpectedly deleted attached ISO media.

## Current Status

Client deployment paused pending clean VM recreation.

---

# Lessons Learned

- Active Directory depends heavily on DNS reachability.
- Bridged networking exposes virtual infrastructure to physical LAN.
- Windows Home editions cannot join domains.
- UEFI/NVRAM virtual machines require special handling during deletion.
- VM install media should be stored outside active libvirt storage paths.
- SSH tunnels can securely expose localhost-bound administrative services.
