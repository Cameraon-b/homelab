### 6. Troubleshot RDP Deployment

Initial symptoms:

- host reachable
- DNS functional
- Group Policy appeared applied
- RDP connection failed

Diagnostic steps:

Validated:

- `ping`
- `gpresult /r /scope computer`
- firewall rules
- Group Policy scope

Observed:

No listener present on port 3389.

Checked:

`TermService`

Result:

Service was already running.

Investigated:

- security filtering
- OU placement
- local group membership
- listener state

Key realization:

A GPO can appear applied while individual settings fail silently due to security filtering or incomplete policy application.

Remediation:

- corrected GPO security filtering
- forced Group Policy update
- rebooted workstation
- verified local group population

Result:

RDP listener successfully initialized.

### 8. Compared Remote Management Protocols

Current remote access methods:

#### TigerVNC

Characteristics:

- console-level access
- independent of domain authentication
- useful during boot or domain failures

#### Remote Desktop (RDP)

Characteristics:

- operating system-level sessions
- Kerberos authentication
- policy-driven access
- role-based security

Key realization:

Multiple remote management paths provide operational resilience.

If domain services fail:

- RDP may fail
- console access remains available
